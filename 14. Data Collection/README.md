## **ðŸ”¹ 1. Databases (SQL & NoSQL)**

Databases are the most structured and reliable data sources for organizations.

### **SQL Databases**

SQL databases follow a **relational model** (tables, rows, columns).
Examples:

- MySQL
- PostgreSQL
- Oracle
- SQL Server

**Characteristics:**

- Follow a predefined schema
- ACID properties (safe & consistent)
- Best for transactional systems

**What type of data you get:**
Customer records, sales data, transactions, user activity logs, etc.

---

### **NoSQL Databases**

Used when data is semi-structured or unstructured.
Examples:

- MongoDB
- Cassandra
- Firebase
- DynamoDB

**Characteristics:**

- Flexible schema
- High scalability
- Stores documents, key-value, graphs, or wide-column data

# **ðŸ“˜ SQL Basics (with Examples)**

---

## **1. Creating a Database**

```sql
CREATE DATABASE company;
```

---

## **2. Creating a Table**

```sql
CREATE TABLE employees (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT,
    salary FLOAT,
    department VARCHAR(50)
);
```

---

## **3. Inserting Data**

```sql
INSERT INTO employees VALUES
(1, 'Amit', 28, 45000, 'IT'),
(2, 'Neha', 32, 52000, 'HR');
```

---

## **4. Selecting Data**

```sql
SELECT * FROM employees;
```

Selecting specific columns:

```sql
SELECT name, salary FROM employees;
```

---

## **5. Filtering Data (WHERE Clause)**

```sql
SELECT * FROM employees WHERE salary > 50000;
```

---

## **6. Sorting Data**

```sql
SELECT * FROM employees ORDER BY salary DESC;
```

---

## **7. Updating Records**

```sql
UPDATE employees
SET salary = 60000
WHERE id = 2;
```

---

## **8. Deleting Records**

```sql
DELETE FROM employees WHERE id = 1;
```

# **ðŸ“Š Advanced SQL Concepts**

---

## **1. Aggregate Functions**

```sql
SELECT AVG(salary), MAX(salary), MIN(salary) FROM employees;
```

---

## **2. GROUP BY**

```sql
SELECT department, AVG(salary)
FROM employees
GROUP BY department;
```

---

## **3. JOINs (VERY Important!)**

### **INNER JOIN**

```sql
SELECT e.name, d.department_name
FROM employees e
INNER JOIN departments d
ON e.department = d.id;
```

### **LEFT JOIN**

```sql
SELECT * FROM employees
LEFT JOIN departments
ON employees.department = departments.id;
```

---

## **4. Subqueries**

```sql
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);
```

---

## **5. SQL for ML Feature Engineering**

- Handling missing values
- Encoding categorical data
- Generating statistical summaries
- Creating new features via SQL expressions

Example:

```sql
SELECT name,
       salary,
       salary / 12 AS monthly_salary
FROM employees;
```
